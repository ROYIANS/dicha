# Zed 官网 section 框架标准代码（用户提供）

来源：zed.dev 官网（zed.html 抓取），一个标准 section 的完整结构。这是本任务的对齐基准。

## 关键结构要点（解读）

1. **section 整体**：`relative flex min-w-0 outer-section-node-offset min-h-[448px]`
   - 横向 flex：`[左侧窄 rail] [左侧弹性 rail (lg+)] [中央 container] [右侧弹性 rail (lg+)] [右侧窄 rail]`
2. **菱形节点（diamond node）**：
   - `absolute z-99 size-1.5 rotate-45 border border-offgray-100 bg-white`
   - 定位：`[bottom:calc(-1*var(--node-vertical-offset))] [left:var(--node-horizontal-offset)]`
   - section 外层有一对（左右），中央 container 上还有一对（`[--node-horizontal-offset:-3.5px]`，lg 才显示）
   - 即：节点中心正好骑在水平分界线上（bottom 为负的 offset 把它往下推一半），水平方向中心正好压在竖线上（-3.5px = 半个 6px 节点 - 0.5px 线宽）
3. **外侧窄 rail**：`relative z-1 w-4 sm:w-6 md:w-12 shrink-0 border-r lg:border-r-0 border-transparent`
   - 内部用 `width:10px; right:-0.5px; transform:translateX(50%)` 的容器 + `width:1px` 的分段竖线（实线/虚线段随机分段，flex 数值随机）
   - 竖线压在 rail 的右边缘（即与中间区域的分界处）
4. **弹性 rail（lg+ 显示）**：`hidden flex-1 lg:block border-x border-transparent`
   - 同样的分段竖线，靠近 container 一侧（right/left ±0.5px）
   - 额外有一个 32px 宽的 SVG pattern（随机长短的水平刻度线 rect，宽 8~24px，间隔 ~7-11px），贴着 container 边缘往外长
5. **中央 container**：`relative container-max-w max-md:min-w-0 flex-1 [--node-horizontal-offset:-3.5px]`
   - 自带左右菱形节点（lg 显示）
   - 内容区：`relative isolate overflow-clip size-full ...` 内含 10x10 网格 SVG 背景
